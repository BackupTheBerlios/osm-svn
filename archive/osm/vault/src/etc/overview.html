Package Description

The net.osm.vault package provides support for the establishment of an authenticated X509 principal. The implementation leverages the JAAS pluggable authentication services to establish an X509 principal based on a keystore, alias and password.

Components
{@link net.osm.vault.DefaultVault} Vault implementation that provides support for the establishment of a authenticated principal.
{@link net.osm.vault.Demonstration} Composite sample component that declares a dependecy on a Vault. The component is supplied with the Vault under the Serviceable phase, following which the Vault is resolved during Initialization. The component invokes a login, and recordes the name of the established X509 principal as an INFO log entry under the Executable phase.

Login Configuration

JASS will attempt to locate the login configration based on the following sequence:

  • lookup all login.config.url.<n> entries in the file <JAVA_HOME>/lib/security/java.security
  • check the System property java.security.auth.login.config
  • if the above fails, JAAS will attempt to locate a login configuration under a file named ${user.home}/.java.login.config

Example Login Configuration

The following is an example of a Windows (TM) NT4 login configuration file named .java.login.config in the user's home directory. Note that URL values entered in the following configuration block must be quoted. The values assigned to keyStorePasswordURL and privateKeyPasswordURL are URLs that refer to password values. 

net.osm.vault.Vault
{
   com.sun.security.auth.module.KeyStoreLoginModule required 
   keyStoreURL="file:C:/WINNT/Profiles/fred/.keystore"
   keyStoreAlias=fred
   keyStorePasswordURL="file:C:/WINNT/Profiles/fred/.password"
   privateKeyPasswordURL="file:C:/WINNT/Profiles/fred/.private"
   debug=false;
};
